splica: Browser-Native Video Demuxing

This demo runs entirely in your browser — no server, no ffmpeg, no native code. The splica WASM module demuxes video files and feeds packets directly to the browser's WebCodecs API for decoding.

Bundle Size Comparison

splica-wasm:
~200 KB (gzipped)
ffmpeg.wasm:
~30 MB (gzipped)

How It Works

ffmpeg.wasm

// 30 MB download
import { FFmpeg } from '@ffmpeg/ffmpeg';
const ffmpeg = new FFmpeg();
await ffmpeg.load(); // downloads 30 MB WASM

// Write file to virtual filesystem
ffmpeg.writeFile('input.mp4', data);

// Shell out to ffmpeg CLI in WASM
await ffmpeg.exec([
  '-i', 'input.mp4',
  '-f', 'rawvideo',
  'output.raw'
]);

// Read result from virtual filesystem
const result = ffmpeg.readFile('output.raw');

// ~200 KB download

await init(); // loads ~200 KB WASM

The Point

ffmpeg.wasm is a full ffmpeg compiled to WebAssembly — ~30 MB of download before your page can process a single frame. It exposes a CLI interface through a virtual filesystem, because that's how ffmpeg was designed: as a command-line tool.

splica-wasm is purpose-built for the browser. ~200 KB, loads instantly, and gives you a native JavaScript API that feeds directly into WebCodecs. No virtual filesystem. No CLI emulation. No 30 MB download.

This is not a feature ffmpeg can add. It's a consequence of designing for the browser from day one instead of porting a 24-year-old command-line tool.
